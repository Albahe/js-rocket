Hello! Here's my Interactive Element

I’m Alba Huguet, applying for the Creative Design Analyst position at Smadex. As part of my application, I’ve completed the task of creating an interactive element that transforms between a red square and a yellow circle, moving between two positions when clicked.

How was this transformation achieved?

This transformation was achieved using HTML, CSS, and JavaScript. Along the way, I encountered several challenges, but in the following section, I will explain the process step by step, highlighting the problems I faced and how I resolved them, accompanied by images for better understanding.

1. First, I created the HTML structure for the page that would hold the transformation, linking it to the corresponding CSS and JavaScript files. I then added an anchor (<a>) element with a <div> inside it, which would represent the red square. I set up an onclick event on the anchor, so that when the square was clicked, it would trigger the function that I would later define in the JavaScript file. To prevent the anchor from reloading the page (which would cause errors), I added return false.

The <div> inside the anchor was initially styled as a red square, using a CSS class called RedSquare. The <div> also had an id of idObject, which would be used later in the JavaScript function to target the element.

2. Next, I wrote the initial CSS to define the color, shape, and proportions of the red square (.RedSquare), as well as its future transformation into a yellow circle (.RedSquare.YellowCircle). At this point, I only focused on styling the visual aspects, such as the color and shape, without specifying the position of the elements on the page. I decided to handle the positioning later, once the JavaScript function was working correctly.

3. Writing the JavaScript function was a challenging but essential task to make the transformation between the red square and the yellow circle work. The function, switchClass(), begins by creating a variable (currentClassName) that retrieves the current className of the element with the ID idObject. This is done using document.getElementById("idObject").className.

Next, an if statement checks if the current class of the element is "RedSquare" (the CSS class defined earlier). If it is, the class is changed to "RedSquare YellowCircle", triggering the transformation into a yellow circle. An else statement checks if the current class is anything other than "RedSquare" (in this case, it would be the yellow circle), so the class is reset to "RedSquare", reverting the element back to the original red square. Now that the transformation is functional, the final task was to write the CSS to change the position of the element and make it move.

4. Through some trial and error, I was able to create a transition similar to the one provided in the task. To achieve this, I adjusted the margin properties to set the initial position of the shapes and specified the duration of the transition.

At first, the element didn’t move diagonally as expected, but I resolved this by using the float property, which corrected the movement.

5. However, I wasn't satisfied with the result because the element wasn't centered on the page, nor was it responsive when the screen size changed. To address this, I decided to place the element inside a wrapper <div>, which I later styled with CSS to ensure proper alignment and responsiveness.

6. After some more trial and error, I focused on centering the .box <div> on the page. To achieve this, I set the body properties to ensure the <div> was centered horizontally. Then, I applied specific styles to the .box class: I defined its height and width, added a black border to visualize it, and set its position to relative, which would be essential for positioning the square and circle within it.

Positioning was crucial to make the red square move from the top-left corner of the <div> to the bottom-right corner. I adjusted the properties of the .RedSquare and .RedSquare.YellowCircle classes by removing the margins and specifying the position as absolute along with top and left values to control the placement.

One challenge I encountered was the yellow circle going outside the bounds of the <div>, particularly when resizing the screen. The solution came when I applied the transform property to shift the anchor point of the yellow circle to the bottom-right corner, ensuring it stayed within the <div>.

Once I was satisfied with the behavior, I made the final touch by setting the border of the .box <div> to transparent, removing the visual border. And voilà! The interactive transformation was successfully completed.
